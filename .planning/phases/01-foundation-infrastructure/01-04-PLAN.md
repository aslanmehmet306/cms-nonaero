  <name>Task 1: Audit trail module with interceptor for automatic entity change logging</name>
  <files>
    apps/api/src/audit/audit.module.ts,
    apps/api/src/audit/audit.service.ts,
    apps/api/src/audit/audit.controller.ts,

  </files>
  <action>
    1. **audit.decorator.ts**: Create @Audit(entityType: string) decorator using SetMetadata. Export AUDIT_KEY constant and Audit decorator.

    2. **audit.service.ts**: Injectable service with:
       - **log(params)**: Create AuditLog record via Prisma with: action (CREATE/UPDATE/DELETE), entityType, entityId, actor (from CLS userId), airportId (from CLS), previousState (JSONB), newState (JSONB), metadata (optional). Timestamp auto-set via @default(now()).
       - **findAll(query)**: Paginated query with filters: entityType, entityId, actor, dateFrom, dateTo. Return with meta (page, perPage, total, totalPages).
       - **findByEntity(entityType, entityId)**: Get full audit history for a specific entity, ordered by createdAt DESC.
       - Import ClsService from nestjs-cls for actor/airportId context.
       - IMPORTANT: Exclude sensitive fields from snapshots (passwordHash, refreshToken). Create a sanitize helper that strips these fields before logging.

    3. **audit-log.interceptor.ts**: NestJS interceptor implementing NestInterceptor.
       - In intercept(): Check for @Audit metadata via Reflector. If no entityType, pass through.
       - Determine action from HTTP method: POST=CREATE, PUT/PATCH=UPDATE, DELETE=DELETE. GET = skip.
       - For UPDATE/DELETE: Before the handler executes, fetch the existing entity state (need entityId from request params). Store as "before" state.
       - After handler returns (via tap in RxJS pipe): Log the audit entry with before state and response data as after state.
       - Handle errors gracefully (audit logging failure should NOT break the request).
       - Use try/catch around audit logging -- log warning if audit fails but don't throw.

    4. **audit.controller.ts**: Endpoints at /api/v1/audit:
       - GET /api/v1/audit -- List audit logs with pagination and filters. @Roles(UserRole.SUPER_ADMIN, UserRole.AIRPORT_ADMIN, UserRole.AUDITOR, UserRole.FINANCE). @Public() = NO (requires auth).
       - GET /api/v1/audit/entity/:entityType/:entityId -- Get audit trail for specific entity. Same role restriction.
       - All endpoints have @ApiTags('audit') Swagger tag.

    5. **query-audit.dto.ts**: entityType (optional string), entityId (optional string), actor (optional string), dateFrom (optional ISO string), dateTo (optional ISO string), page (optional number, default 1), perPage (optional number, default 25). All with @ApiPropertyOptional() decorators.

    6. **audit.module.ts**: Provide AuditService, AuditLogInterceptor. Export AuditService, AuditLogInterceptor.

    7. **Update app.module.ts**: Import AuditModule.
  </action>

    2. **health.controller.ts**: Controller at /health prefix. Both endpoints are @Public() (no auth required).
       - **GET /health/liveness**: Lightweight check. Use HealthCheckService with MemoryHealthIndicator.checkHeap('memory_heap', 300 * 1024 * 1024). Returns 200 if process is alive and memory is under threshold.

         - Database check: PrismaService.$queryRaw`SELECT 1` wrapped in a custom check function (Terminus PrismaHealthIndicator may not exist, so use custom).

         - Returns 200 when all dependencies reachable, 503 ServiceUnavailable when any fails.
