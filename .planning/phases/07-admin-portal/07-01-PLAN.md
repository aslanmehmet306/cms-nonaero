    import ReactDOM from 'react-dom/client';
    import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

    import { Toaster } from 'sonner';

  <action>
    **1. Create Zustand auth store** (`src/store/authStore.ts`):
    - Interface: `{ accessToken, refreshToken, user: { sub, email, role, airportId? }, login(), logout() }`
    - Use `persist` middleware with `name: 'auth-storage'` for localStorage persistence
    - `logout()` must clear tokens AND call `queryClient.clear()` to wipe TanStack Query cache (Pitfall 7)
    - Import queryClient from main.tsx for cache clearing

    **2. Create Axios API client** (`src/api/client.ts`):
    - `baseURL: '/api/v1'`
    - Request interceptor: reads token from `useAuthStore.getState().accessToken` (NOT a hook call — outside React)
    - Response interceptor: on 401, call `useAuthStore.getState().logout()` and redirect to `/login`

    **3. Create auth API module** (`src/api/auth.ts`):
    - `adminLogin(email, password)` → POST `/auth/admin/login` → returns `{ access_token, refresh_token, expires_in, user }`
    - `refreshToken(refresh_token)` → POST `/auth/admin/refresh`
    - `getMe()` → GET `/auth/me`

    **4. Create ProtectedRoute** (`src/router/ProtectedRoute.tsx`):
    - Reads `accessToken` from Zustand
    - If no token → `<Navigate to="/login" replace />`
    - If token → `<Outlet />`

    **5. Create router** (`src/router/index.tsx`):
    - Use `createBrowserRouter` with two top-level branches (Pitfall 5):
      - Public: `<Route path="/login" element={<Login />} />` — NO AppShell wrapper
      - Protected: `<Route path="/" element={<ProtectedRoute />}> <Route element={<AppShell />}> ...feature routes... </Route> </Route>`
    - Feature routes (render placeholder divs for now, actual pages come in 07-02 and 07-03):
      - `/` → Dashboard (placeholder)
      - `/contracts` → ContractList (placeholder)
      - `/contracts/new` → ContractForm (placeholder)
      - `/contracts/:id` → ContractDetail (placeholder)
      - `/tenants` → TenantList (placeholder)
      - `/tenants/new` → TenantForm (placeholder)
      - `/tenants/:id` → TenantForm (placeholder, edit mode)
      - `/formulas` → FormulaList (placeholder)
      - `/formulas/:id` → FormulaBuilder (placeholder)
      - `/services` → ServiceList (placeholder)
      - `/billing` → BillingList (placeholder)
      - `/invoices` → InvoiceList (placeholder)
      - `/settings` → Settings (placeholder)
    - Each placeholder: `<div className="p-6"><h1 className="text-2xl font-bold">{Page Name}</h1><p className="text-muted-foreground">Coming soon</p></div>`

    **6. Create Login page** (`src/pages/Login.tsx`):
    - Centered card layout using Shadcn Card component
    - Form with email and password inputs using react-hook-form + zod validation
    - On submit: call `adminLogin()`, on success call `authStore.login(access_token, user)` and `navigate('/')`
    - Show error toast via sonner on invalid credentials
    - Title: "Airport Revenue Management" subtitle: "Admin Portal Login"

    **7. Create AppShell** (`src/components/layout/AppShell.tsx`):
    - Flex layout: Sidebar (fixed width ~256px) + main content area
    - Main area: Header at top + `<Outlet />` for route content with padding
    - Responsive: min-width: 1366px (R12.10 desktop-first)

    **8. Create Sidebar** (`src/components/layout/Sidebar.tsx`):
    - Navigation links using Shadcn components with lucide-react icons:
      - Dashboard (LayoutDashboard icon)
      - Contracts (FileText icon)
      - Tenants (Building2 icon)
      - Services (Wrench icon)
      - Formulas (Calculator icon)
      - Billing (CreditCard icon)
      - Invoices (Receipt icon)
      - Settings (Settings icon)
    - Active link highlighted based on current route (useLocation)
    - Role-based visibility: auditor sees all links but action buttons will be disabled in feature pages (not the sidebar's concern)
    - App title "ADB Revenue" at top of sidebar

    **9. Create Header** (`src/components/layout/Header.tsx`):
    - Right-aligned user info: email from Zustand + role badge
    - Logout button (calls authStore.logout() and navigates to /login)
    - Use Shadcn DropdownMenu for user menu (avatar/initials → dropdown with "Sign out")

    **10. Create shared components:**

    **StatusBadge** (`src/components/shared/StatusBadge.tsx`):
    - Maps all status enum values to Shadcn Badge variants (default/secondary/destructive/outline)
    - Covers: ContractStatus, TenantStatus, BillingRunStatus, InvoiceStatus, ObligationStatus
    - Replaces underscores with spaces, capitalizes display text

    **DataTable** (`src/components/shared/DataTable.tsx`):
    - Generic wrapper around @tanstack/react-table + Shadcn Table component
    - Props: `columns: ColumnDef<T>[]`, `data: T[]`, optional `searchKey`, optional `pageSize` (default 10)
    - Built-in: column sorting, pagination (prev/next buttons), optional text filter
    - Uses `useReactTable` with `getCoreRowModel`, `getPaginationRowModel`, `getSortedRowModel`

    **ConfirmDialog** (`src/components/shared/ConfirmDialog.tsx`):
    - Wraps Shadcn AlertDialog
    - Props: `title`, `description`, `onConfirm`, `variant?: 'default' | 'destructive'`, `trigger` (ReactNode)
    - Used for: state transitions, deletions, approvals

    **PageHeader** (`src/components/shared/PageHeader.tsx`):
    - Props: `title: string`, `description?: string`, `actions?: ReactNode`
    - Layout: title left-aligned, actions right-aligned, description below title
    - Used at top of every feature page

    **11. Create useSSE hook** (`src/hooks/useSSE.ts`):
    - `useBillingSSE(billingRunId: string | null)` — opens EventSource to `/api/v1/billing-runs/${id}/progress?token=${token}`
    - Token read from `useAuthStore.getState().accessToken`
    - Returns progress state object, updates on each SSE message
    - Cleanup: closes EventSource on unmount or billingRunId change
    - Also create `useNotificationSSE()` variant for notification bell (connects to `/api/v1/notifications/sse?token=${token}`)
