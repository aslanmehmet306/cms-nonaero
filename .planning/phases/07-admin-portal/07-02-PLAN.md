<objective>
Build the four primary admin feature areas: Contract management (list + detail + create/edit with state transitions), Tenant management (list + create/edit with status lifecycle), Formula/Service management (list + formula builder with dry-run), and Billing operations (list + trigger modal with real-time SSE progress + approve/reject).

Purpose: These are the core admin workflows the demo audience will interact with. Contracts and billing are the most complex — they have state machines, sub-entity assignments, and real-time progress tracking. This plan delivers all four together because they share the same API patterns (TanStack Query + DataTable + StatusBadge).

Output: Four fully functional feature areas replacing route placeholders from 07-01.
