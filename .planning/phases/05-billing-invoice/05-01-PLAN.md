    **Step 9: Create billing.service.ts**

    Inject PrismaService, EventEmitter2, and `@InjectQueue('billing-run') billingRunQueue: Queue`.

    Methods:
    - `createBillingRun(dto)`: Enforce concurrency rule R8.7 — findFirst where airportId+periodStart+periodEnd AND status NOT IN terminal statuses. If found, throw ConflictException. Create BillingRun with status=initiated, filters={tenantIds: dto.tenantIds || []}. Add job to queue: `billingRunQueue.add('process-billing-run', { billingRunId }, { jobId: billingRunId, attempts: 3, backoff: { type: 'exponential', delay: 5000 } })`. Return run.

    - `transitionRun(id, toStatus, opts?)`: Use validateBillingRunTransition. Update BillingRun status. If toStatus is cancelled, set completedAt. If approved, set approvedBy+approvedAt from opts. Emit 'billing.progress' event.

    - `scopeObligations(billingRunId)`: Load billing run. Get tenantIds from filters (if empty, scope all tenants for airport). Query obligations where airportId matches, periodStart/periodEnd match, status='ready', and optionally filter by tenantIds. For delta mode (runMode='delta'), exclude obligations where invoiceLogId IS NOT NULL. Update billingRun.totalObligations with count. Link obligations to billingRun by updating billingRunId. Return obligation list.

    - `createContractSnapshot(billingRunId)`: Load billing run with obligations -> contracts. Build JSONB snapshot of unique contracts (id, contractNumber, version, tenantId, services, areas). Store in billingRun.contractSnapshot.

    - `approveBillingRun(id, dto)`: Validate current status is draft_ready. Transition to approved. Set approvedBy, approvedAt. Add invoice-generation job to a separate 'invoice-generation' queue (placeholder for 05-03).

    - `rejectBillingRun(id)`: Transition to rejected.

    - `cancelBillingRun(id)`: Transition to cancelled. Unlink obligations (set billingRunId=null, keep obligations in 'ready' status so they can be re-run).

    - `findAll(query)`: List billing runs with pagination, filter by airportId, status, period.
    - `findOne(id)`: Get single billing run with obligations count.

    **Step 10: Create billing-run.processor.ts**

    `@Processor('billing-run', { concurrency: 1 })` extending WorkerHost.

    Inject BillingService and EventEmitter2.

    `process(job)` method:
    1. Extract billingRunId from job.data
    2. Transition to 'scoping', emit progress 10%
    3. Call createContractSnapshot
    4. Call scopeObligations, emit progress 30%
    5. Transition to 'calculating'
    6. For each obligation, it is already in 'ready' status (calculated by Phase 4 formula engine). The billing pipeline confirms they are ready. Emit progress incrementally up to 80%.
    7. Transition to 'draft_ready', emit progress 100%
    8. Return result

    `@OnWorkerEvent('failed')` handler: log error, transition run to cancelled with error in errorLog.
