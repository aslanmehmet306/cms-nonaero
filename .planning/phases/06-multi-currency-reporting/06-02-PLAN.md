    1. Create EntityTimelineQueryDto in dto/entity-timeline-query.dto.ts:
       - entityType: string (IsNotEmpty)
       - entityId: string (IsUUID)
       (Simple DTO -- no pagination needed for entity timeline; entity history is typically < 100 entries.)

    2. Add diffStates() method to AuditService:
       ```typescript
       diffStates(
         previousState: Record<string, unknown> | null,
         newState: Record<string, unknown> | null,
       ): Array<{ field: string; from: unknown; to: unknown }>
       ```
       - If previousState is null (CREATE): return all newState fields as {field, from: null, to: value}
       - If newState is null (DELETE): return all previousState fields as {field, from: value, to: null}
       - Otherwise: compare each key, include only fields where JSON.stringify(prev[key]) !== JSON.stringify(new[key])
       - Skip internal fields: 'updatedAt', 'createdAt' (noisy, not useful for drill-down)

    3. Add getEntityTimeline() method to AuditService:
       ```typescript
       async getEntityTimeline(entityType: string, entityId: string): Promise<EntityTimelineResponse>
       ```
       - Fetch audit entries via findByEntity(entityType, entityId) (reuse existing method)
       - Map each entry to: { timestamp: createdAt, action, actor, changes: diffStates(previousState, newState) }
       - Enrich with domain context based on entityType:
         * 'Obligation': Query prisma.obligation.findUnique({ where: { id: entityId }, select: { calculationTrace: true, status: true, amount: true, currency: true, chargeType: true } }). Include calculationTrace in response.
         * 'Contract': Query prisma.contract.findUnique({ where: { id: entityId }, select: { contractNumber: true, version: true, status: true, _count: { select: { obligations: true } } } }). Include obligationCount and contract summary.
         * All other entity types: No domain enrichment, just audit trail with diffs.
       - Return { entityType, entityId, enrichment: { ...domain context or null }, timeline: [...entries] }

    4. Inject PrismaService into AuditService constructor (it already has it -- just use it for the new queries).

    5. Create unit tests with mocked PrismaService. Mock auditLog.findMany, obligation.findUnique, contract.findUnique. Test all behaviors above.
