    2. **Add parseAndValidateUpload method** to DeclarationsService:
    ```typescript
    async parseAndValidateUpload(
      file: Express.Multer.File,
      airportId: string,
    ): Promise<{ created: number; errors: Array<{ row: number; field: string; rule: string; message: string }> }>
    ```
    Steps:
    a. Detect file type from mimetype: CSV (`text/csv`) or Excel (`application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`, `application/vnd.ms-excel`)
    b. Parse into rows: CSV via `buffer.toString('utf-8').split('\n').map(r => r.split(','))` with header row detection; Excel via `XLSX.read(buffer, { type: 'buffer' })` then `XLSX.utils.sheet_to_json()`
    c. Expected columns: `tenantId, contractId, periodStart, periodEnd, category, grossAmount, deductions`
    d. Apply 6 validation rules per row:
       - MISSING_FIELDS: tenantId, periodStart, grossAmount required
       - INVALID_TENANT: prisma.tenant.findUnique check (batch all tenantIds first for efficiency)
       - INVALID_PERIOD: Date.parse check on periodStart
       - NEGATIVE_AMOUNT: parseFloat(grossAmount) < 0
       - DUPLICATE_PERIOD: Set-based check within uploaded rows (tenantId:periodStart:category)
       - DEVIATION_THRESHOLD: query previous period declaration for same tenant+category, flag if |current - previous| / previous > 0.30 (30% default). This is a WARNING, not a rejection — include in errors array with rule='DEVIATION_THRESHOLD' but still create the declaration.
    e. For valid rows: Create Declaration (status=draft) + DeclarationLine in a transaction
    f. Return `{ created, errors }` summary

    3. **Add upload endpoint** to DeclarationsController:

    ```typescript
    @Post('upload')
    @UseInterceptors(FileInterceptor('file', {

        cb(null, allowed.includes(file.mimetype));
      },
    }))
    ```
